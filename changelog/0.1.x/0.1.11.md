# 0.1.11 — 2026-06-04

## Fixed

- **`eurostat_browse_themes`** — handler now catches `not_found` from the service and re-throws via `ctx.fail`, populating `data.recovery.hint` with navigation guidance back to root themes ([#10](https://github.com/cyanheads/eurostat-mcp-server/issues/10)).
- **`eurostat_get_dataset_info`** — handler catches `not_found` and `async_response` and re-throws via `ctx.fail` with contextual recovery hints (search suggestion; retry note for the unexpected async path) ([#10](https://github.com/cyanheads/eurostat-mcp-server/issues/10)).
- **`eurostat_get_dimension_values`** — handler catches `not_found` and re-throws via `ctx.fail` with a hint pointing to `eurostat_search_datasets` and `eurostat_get_dataset_info` ([#10](https://github.com/cyanheads/eurostat-mcp-server/issues/10)).
- **`eurostat_query_dataset`** — handler catches all four declared error reasons (`no_results`, `async_response`, `not_found`, `invalid_dimension`) and re-throws via `ctx.fail` with dataset-contextual recovery hints; `conflicting_params` continues to fall through ([#11](https://github.com/cyanheads/eurostat-mcp-server/issues/11)).
- **Tests** — updated and extended to assert `data.recovery.hint` is populated on every error path; added new test cases for `async_response` in `eurostat_get_dataset_info` and `invalid_dimension` in `eurostat_query_dataset`.
