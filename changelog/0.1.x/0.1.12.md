# 0.1.12 — 2026-06-15

## Added

- **Server identity** — `createApp()` now sets `name` and `title` to `eurostat-mcp-server`, pinning the display identity to the unscoped repo name on every client surface ([core #213](https://github.com/cyanheads/mcp-ts-core/issues/213)).
- **`Dockerfile` `HEALTHCHECK`** — bun-native `fetch` against `/healthz` (slim image ships no curl/wget), plus `org.opencontainers.image.version` stamped from the `APP_VERSION` build arg.
- **`scripts/clean-mcpb.ts`** — post-pack step that strips dependency-shipped agent docs (`node_modules/**/skills/`, `.claude/`, `.agents/`, `SKILL.md`) from the `.mcpb` bundle; wired into the `bundle` script ([core #230](https://github.com/cyanheads/mcp-ts-core/issues/230)).

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.21` → `^0.10.6` — env booleans now parse via Zod `stringbool` (rejects values outside `true/false/1/0/yes/no/on/off`); DataCanvas SQL gate fails closed on non-SELECT and denies `pragma_*`/system-catalog table functions; wire-level elicitation + URL-mode elicitation; `outlineOnOverflow()` and `ctx.enrich.truncated()` utilities; `RootsRegistry` and the sampling surface removed. No server code depends on the removed surfaces.
- **`.mcpbignore`** — root dev-dir patterns anchored with `/` (`/skills/`, `/.agents/`, `/.claude/`) so they no longer also strip nested runtime paths like `node_modules/x/skills/` ([core #172](https://github.com/cyanheads/mcp-ts-core/issues/172), [#207](https://github.com/cyanheads/mcp-ts-core/issues/207)).
- **`scripts/lint-packaging.ts`** — adds bundle-content guards (unanchored-pattern and critical-runtime-path checks, post-bundle `node_modules` agent-doc scan) and an entrypoint identity check that `name`/`title` equal the unscoped package name ([core #230](https://github.com/cyanheads/mcp-ts-core/issues/230), [#231](https://github.com/cyanheads/mcp-ts-core/issues/231)).
- **`scripts/check-framework-antipatterns.ts`** — adds the `coerce-boolean-env-flag` rule flagging `z.coerce.boolean()` on env flags (use `z.stringbool()`), and skips comment-line matches.
- **Skills** — re-synced from the framework; adds the `techniques` skill (outline-on-overflow pattern catalog).
