# 0.1.13 — 2026-06-20

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.10.6` → `^0.10.9` — `DuckdbProvider.describe({ tableName })` qualifies its `WHERE` filters with the table alias, fixing the DuckDB Binder Error (ambiguous `table_name`) that hit every filtered describe ([core #235](https://github.com/cyanheads/mcp-ts-core/issues/235)); the Canvas SQL gate now classifies a `SELECT`-shaped statement that parses but fails to prepare (mistyped column, unknown function) as `ValidationError` with `data.reason: 'invalid_sql'` and the DuckDB binder detail in `data.binderMessage`, instead of the misleading `non_select_statement` ([core #236](https://github.com/cyanheads/mcp-ts-core/issues/236)); adds `ctx.content` — an always-present collector for image/audio blocks that ride `content[]` only and never enter `structuredContent` ([core #239](https://github.com/cyanheads/mcp-ts-core/issues/239)); corrects `OTEL_SERVICE_NAME` seeding precedence (`createApp({ name })` seeds it only when unset) ([core #233](https://github.com/cyanheads/mcp-ts-core/issues/233)); biome `2.4.16` → `2.5.0` and fresh-scaffold devcheck guards that skip git-dependent checks cleanly when `.git` is absent ([core #243](https://github.com/cyanheads/mcp-ts-core/issues/243)).
- **`scripts/check-dependency-specifiers.ts`** — new devcheck step (`Dependency Specifiers`, `--no-dep-specifiers`) that hard-fails on floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json`'s dependency sections and `bun.lock`'s `workspaces` map; catches `bun update --latest` writing a literal `latest` dist-tag into the lock's workspace map, which would re-resolve the dep past the `package.json` range on the next install ([core #246](https://github.com/cyanheads/mcp-ts-core/issues/246)).
- **`scripts/lint-packaging.ts`** — adds plugin-marketplace manifest checks (check 10) validating `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json`: non-empty descriptions, the unscoped machine name in display fields, and the full scoped name in the `npx -y` install arg (an unscoped arg for a scoped package is a guaranteed install 404) ([core #240](https://github.com/cyanheads/mcp-ts-core/issues/240)).
- **`devcheck.config.json`** — enables `packaging.pluginManifests`, running the Packaging gate against the plugin manifests now that this server ships them.
- **Skills** — re-synced from the framework; seven `metadata.version` bumps propagate body drift (`api-auth`, `api-errors`, `api-services`, `api-telemetry`, `field-test`, `report-issue-local`, `tool-defs-analysis`) ([core #238](https://github.com/cyanheads/mcp-ts-core/issues/238)).
